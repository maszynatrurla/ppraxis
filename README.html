1. RPN calculator.

Command line calculator using reverse polish notation.

Inspiration: http://programmingpraxis.com/2009/02/19/rpn-calculator/
Build: make rpn
Run: ./bin/rpn

Calculator supports operations on integers and floating point numbers. Precision is limited to that of float type. Available operations: + (addition), - (substraction), * (multiplication), / (division). Numbers use decimal notation, use period or comma as decimal point and blanks (e.g. spaces) as token separators. Type 'q' to quit. Example usage:
Input: >19 2.14 + 4.5 2 4.3 / - *
Output: 

    plus(19, 2.14) = 21.14
    div(2, 4.3) = 0.465116
    minus(4.5, 0.465116) = 4.03488
    mul(21.14, 4.03488) = 85.2974

    =    85.2974

11. Mark V. Shaney.

Generates paragraphs of text that almost makes sense.
Inspiration: http://programmingpraxis.com/2009/02/27/mark-v-shaney/
Build: make shaney
Run: ./bin/shaney COMMAND [FILENAME] ...

Program is able to generate short paragraphs based on vocabulary taken from teaching texts. The results will look like a parodies of material given as input. Available commands:
use FILENAME - specify database file to use; when used together with learn command, it will append learned vocabulary to file; when used together with write command, it will use vocabulary from database file to generate text;
learn FILENAME - learn vocabulary from the text file with given name; the result of text analysis can then be saved to file or/and used immediately to generate text;
write - generate text; must be used together with command learn, use or both.

Examples:
$ bin/shaney learn teksty.txt write
Analyse vocabulary from file teksty.txt and generate paragraph of text with similar style.

$ bin/shaney use baza.db learn teksty.txt
Analyse vocabulary from file teksty.txt and save analysis result to file baza.db. If the file aleady exists, the results will be appended to existing base. Otherwise, new file will be created.

$ bin/shaney use baza.db write
Read vocabulary from file baza.db and use it to generate paragraph of text.

19. Turing Machine Simulator.

Simulation of Turing-like machine.
Inspiration: http://programmingpraxis.com/2009/03/27/a-turing-machine-simulator/
Build: make turing
Run: ./bin/turing TAPE_FILE CODE_FILE

The program takes as input two files: one has initial contents of Turing machine tape; the other has Turing machine code in form of transition table.
The tape file is a text file, with contents of tape cells separated by whitespaces. Special 'blank' symbol is written as underscore (_). Initial position of Turing machine pointer (rw head) has to be specified using square braces e.g. 

_ [1] 1 0
Example tape file can be found in dat/turingTape.
The code file has to contain table of transitions. Each transition is made of 5 elements separated by whitespaces: (1) current state, (2) symbol which was read, (3) symbol to write, (4) move direction, (5) next state to enter. The states (1, 5) must be integer numbers. Symbols (2, 3) are arbitrary ASCII characters. Direction (4) must be either 'R' (move pointer right) or 'L' (move pointer left).
Example code file can be found in dat/turingCode.
Machine starts in state 0. In each step it looks into transition table to find what should be done in current situation (state and symbol at pointer). When state becomes a negative number, machine will stop.
Program outputs a final contents of tape.

37. Double transposition cipher.

Encrypts or decrypts text using two-pass of columnar cipher.
Inspiration: http://programmingpraxis.com/2009/05/29/double-transposition-cipher/
Build: make columnar
Run: ./bin/columnar encrypt|decrypt PWD1 PWD2

The program allows to encrypt the text using columnar cipher twice: first time with PWD1 password, second time with PWD2 password (the result is harder to break than single pass of the cipher, especially if password lengths are different). The program also allows to decipher the text. In this case it is important to remember that the passwords should be used in reversed order than during the encryption.
Program takes three command line arguments: first one chooses operation mode (encrypt for encryption, decrypt for decryption); second and third are the passwords to use. For example
./bin/columnar encrypt ciap kotlet
./bin/columnar decrypt kotlet ciap
Notice that the second execution is able to decrypt text encoded during first execution.
During execution program will accept text given as standard input and encrypting/decrypting it line by line. Type Ctrl + D (EOF) to exit.

38. Pig latin.

Mangles words using the rules of children game "pig-latin".
Inspiration: https://programmingpraxis.com/2009/06/02/pig-latin/
Build: make piglatin
Run: ./bin/piglatin encode|decode

The program mangles or unmangles the words given to standard input by applying the rules for chldren English language game "pig-latin".
If encode is passed as a first argument, the program will try to translate each word passed to standard input into pig latin. If the word starts with a vowel, then syllabe -ay is added to the end of word. If the word starts with cosonant, then the beginning cosonants are moved to the back of the word together with syllabe -ay.
In decode mode, program will try to reverse the operation - translate pig latin back to English.
Type Ctrl + D (EOF) to exit.

41. Feynmann's puzzle.

Solves long division puzzle, supposedly originally proposed by Richard Feynmann.
Inspiration: https://programmingpraxis.com/2009/06/12/feynmans-puzzle/
Build: make feynmann
Run: ./bin/feynmann

Program tries to find the hidden figures in given long-division example:

             xxAx
          --------
          xxxxAxx / xAx
          xxAA
         ---------
          xxxA
           xxA
         --------
           xxxx
           xAxx
         --------
            xxxx
            xxxx
           ------
            = =
where every A is the same digit and every x is any digit (but never the same as A).
